Detailed Description

The Biceps-Code calls the different libraries, also parses the commandline for parameters.

Author:
Buote Xu (Buote.Xu@stud.uni-heidelberg.de)
Date:
January, 2010 This class represents the glue to unite the different libraries. It works in several steps: At the beginning, initialize(argc, argv) will call the CommandParser, which is responsible for parsing the command-line parameters and changing the settings/flags accordingly. During the next step, the mandatory fasta-database will be indexed and be made ready for later parsing. The other mandatory parameter corresponds to the MGF-file, which is parsed line by line via readMGF . Every time a complete spectrum is parsed (with BEGIN IONS and END IONS lines), it gBiceps passed in a temporary file to Directag/Pepnovo in run_programs. One of these two libraries (or even both, depending on the settings), creates a list of viable tags corresponding to the current spectrum found in the mgf. runPepsplice calls the Fasta class to process these tags and matching them to the original fasta, to create a customized fasta to pass to the pepsplice-library along with the mgf. This will happen several times per spectrum, as different parameters to pepsplice have to be processed if the result wasn't good enough yet. When all of it is finished, the best pepsplice-outputs for every spectrum is saved and written to a file at the end, while cutting out some of them according to a quantile calculated by findCutoff

Example Usage: 

 int main(int argc, char *argv[])
 {
   Biceps(argc,argv);
 }
